## Cross-Phase Contracts (Outgoing)

Per CONTEXT § Cross-Phase Contracts, **the inline-SVG-with-currentColor pattern** established by Plan 1 and validated end-to-end by this plan is now a **codebase idiom** for Phase 6 (Gallery) and Phase 7 (About) to inherit:

- **Pattern:** Use Hugo's `readFile` to inline SVG markup directly into HTML (avoiding `<img>` round-trip and CSS background-image), with `fill="currentColor"` on every fill so a single CSS `color: var(--text)` rule recolors the entire SVG on theme toggle.
- **Use cases:** Any future iconography (gallery navigation chevrons, about-page social glyphs, footer ornaments) that needs theme-responsive recoloring should adopt this pattern instead of the prior PNG-swap approach (Phase 5 v2.0 / pre-05.1).
- **Trade-off acknowledgment:** Inline SVG adds a few hundred bytes per page render but eliminates an HTTP round-trip, prevents the wordmark "flash" on slow connections, and gives precise theme parity for free. For Plan 1's `logo.svg` (~25 KB) the inlining cost is acceptable; for larger photographic assets (gallery photos), the responsive-image pipeline remains the right answer.
