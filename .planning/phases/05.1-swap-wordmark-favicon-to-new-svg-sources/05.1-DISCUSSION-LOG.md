## Wordmark embedding

| Option | Description | Selected |
|--------|-------------|----------|
| Inline SVG + currentColor (Recommended) | Hugo template inlines the SVG markup; replace fill=#000000 with fill=currentColor; CSS color: var(--text) drives both themes. Single asset, no [data-theme] swap, no seam-mask CSS. | ✓ |
| Two <img> swap with two SVG variants | Pre-bake logo-light.svg and logo-dark.svg at build time; keep the existing two-img + display:none swap. | |
| Single <img> + CSS filter | <img src=logo.svg> with filter: invert(1) under [data-theme=dark]. Fragile; any future multi-color brand work breaks. | |
| Hugo's resources.GetMatch + minify-inline | resources.Get + minify + safeHTML. Same end result as Inline SVG + currentColor but goes through Hugo's asset pipeline. | |

**User's choice:** Inline SVG + currentColor

**Notes:** Cascades to deletion of two-image display swap rules and per-variant seam-mask hex backgrounds. Phase 4 D-02 hard cross-phase contract is dissolved by this phase because the underlying problem (PNG corner-pixel mismatch with Flexoki --bg) ceases to exist for SVG paths.

## Favicon SVG approach

| Option | Description | Selected |
|--------|-------------|----------|
| Native SVG + embedded prefers-color-scheme @media (Recommended) | Replace 77 KB PNG-wrapped favicon.svg with native ~3-5 KB SVG. Embedded <style> flips fill via @media (prefers-color-scheme: dark). | ✓ |
| Native SVG + currentColor only | fill=currentColor — but browsers don't apply CSS color in favicon context. Single-color icon only. | |
| Keep PNG-wrapped favicon.svg, just regenerate from new fav-icon.svg | Same 75 KB ballpark but sharper. Ignores that we now have a real path-based SVG. | |
| Single hardcoded color, no theme swap | fill=#100F0F. Loses favicon-respects-OS-dark-mode behavior. | |

**User's choice:** Native SVG + embedded prefers-color-scheme @media

**Notes:** Favicons render in tab chrome outside the DOM, so they cannot read [data-theme]. prefers-color-scheme is the only viable theme-aware selector for favicons — same tradeoff as Phase 5 D-07 (if user toggles in-page theme to dark while OS is light, favicon stays light). This is NOT the same anti-pattern as the wordmark <picture>+prefers-color-scheme rejection because there is no in-page favicon toggle to override.
