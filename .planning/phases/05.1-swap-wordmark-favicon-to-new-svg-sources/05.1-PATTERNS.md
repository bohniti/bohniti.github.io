### `themes/minimal/static/favicon.svg` (static asset, OVERWRITE)

**Analog:** None in repo (current Phase 5 file is PNG-wrapped — fundamentally different construction). Source is `images/brand-source/fav-icon.svg`.

**Source SVG header pattern** (`images/brand-source/fav-icon.svg` lines 1-3):
```svg
<svg version="1.1" id="Layer_1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" x="0px" y="0px"
     width="100%" viewBox="0 0 896 880" enable-background="new 0 0 896 880" xml:space="preserve">
<path fill="#000000" opacity="1.000000" stroke="none"
    d="
M374.468658,881.000000
```
The source contains 3 `fill="#000000"` declarations across 3 `<path>` elements (D-09).

**Required transformation** (D-09, D-10):
```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 896 880">
  <style>
    path { fill: #100F0F; }
    @media (prefers-color-scheme: dark) {
      path { fill: #FFFCF0; }
    }
  </style>
  <path d="..."/>
  <path d="..."/>
  <path d="..."/>
</svg>
```
- Drop the `version`, `id`, `xmlns:xlink`, `x`, `y`, `width="100%"`, `enable-background`, `xml:space` attributes (none of these are required for browser-tab favicon rendering — they bloat the file)
- Drop `fill="#000000"` and `opacity="1.000000"` and `stroke="none"` from each `<path>` — replaced by the embedded `<style>` selector
- Insert the `<style>` block as the first child of `<svg>`
- Final size target: ~3-5 KB (D-08)

**Hardcoded hex justification** (UI-SPEC § Color):
- `#100F0F` = Flexoki ink (matches light-mode `--text` in style.css line 8)
- `#FFFCF0` = Flexoki Paper (matches light-mode `--bg` and is a deliberate variant of dark-mode `--text` `#CECDC3` — favicon uses Paper for brand correctness; cross-file invariant flagged in UI-SPEC)

**Why hex, not `var(--text)`:** `<style>` inside an SVG file referenced as `<link rel="icon">` runs in an isolated CSSOM and cannot read `:root` custom properties from the host document (UI-SPEC § Color). Hardcoded hex is structurally required, not a style choice.
