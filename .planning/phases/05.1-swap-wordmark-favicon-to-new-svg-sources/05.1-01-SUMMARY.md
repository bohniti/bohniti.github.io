# Phase 05.1 Plan 1: Swap wordmark to inline SVG (Summary)

Two-image PNG wordmark swap replaced with a single inlined SVG that recolors via `currentColor` + Phase 4 `var(--text)` token. Eliminates duplicate HTTP fetches, the seam-masking hex backgrounds, the per-theme display-toggle, and the `[data-theme]`-keyed display logic — all replaced by one `readFile | safeHTML` call and one CSS rule.

## What Changed

### NEW: `themes/minimal/static/images/brand/logo.svg` (24,992 bytes, 396 lines)

Generated from `images/brand-source/logo.svg` by:

1. Rewriting the root opening tag from the original two-line form
   ```svg
   <svg version="1.1" id="Layer_1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" x="0px" y="0px"
        width="100%" viewBox="0 0 1024 624" enable-background="new 0 0 1024 624" xml:space="preserve">
   ```
   into the new single-line form
   ```svg
   <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1024 624" width="200" height="122" role="img" aria-label="Timo Bohnstedt">
     <title>Timo Bohnstedt</title>
   ```
   Drops: `version`, `id`, `xmlns:xlink`, `x`, `y`, `width="100%"`, `enable-background`, `xml:space`. Adds: explicit `width="200" height="122"` (CLS gate per UI-SPEC § Verification Gates), `role="img"`, `aria-label="Timo Bohnstedt"`, child `<title>Timo Bohnstedt</title>` (a11y-equivalent to former `alt`).
2. Replacing all 11 occurrences of `fill="#000000"` with `fill="currentColor"`. Source verified: `grep -c 'fill="#000000"' images/brand-source/logo.svg` returned `11`. Output verified: `grep -c 'fill="currentColor"' themes/minimal/static/images/brand/logo.svg` returns `11`, `grep -c 'fill="#000000"' ...` returns `0`.
3. Preserving every `d="..."` payload byte-for-byte across all 11 `<path>` elements. Verified post-write by reading both files into Python, applying the same `fill` substitution to the source body, and confirming string equality of the bodies after the root `<svg>` opening tag.

T-05.1-01 mitigation gate (`grep -ciE '<script|<foreignObject|onload=|onclick=|onerror=' logo.svg`) returns `0`. XML parses cleanly via Python's ElementTree.

## Hugo Path Resolution Notes

`readFile` takes a project-root-relative path. The path `"themes/minimal/static/images/brand/logo.svg"` resolves at build time when Hugo runs from the project root (which both local `hugo` invocations and the GitHub Actions workflow do). No surprises encountered. Hugo will still also serve `themes/minimal/static/images/brand/logo.svg` at the runtime URL `/images/brand/logo.svg` because it lives under `static/`, but no `<img>` tag references it post-edit — the file is only consumed at build time via `readFile` for inlining.

`resources.Get` was an alternative (would let Hugo Pipes participate), but Hugo Pipes static-dir is not currently configured for this theme, so `resources.Get "images/brand/logo.svg"` would return nil at build time. `readFile` was chosen per plan D-01.

Hugo is not installed locally (per CLAUDE.md), so the optional `hugo --minify` smoke test was skipped. The GitHub Actions deploy (`.github/workflows/deploy.yml`, Hugo Extended 0.157.0) will surface any template errors at the next push.
