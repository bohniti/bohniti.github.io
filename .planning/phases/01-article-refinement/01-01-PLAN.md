```go-html-template
<div class="mermaid">
{{ .Inner | safeHTML }}
</div>
{{ if not (.Page.Scratch.Get "mermaidLoaded") }}
  {{ .Page.Scratch.Set "mermaidLoaded" true }}
  <script src="https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js"></script>
  <script>
    mermaid.initialize({
      startOnLoad: true,
      theme: 'base',
      themeVariables: {
        background: '#FFFCF0',
        primaryColor: '#F2F0E5',
        primaryTextColor: '#100F0F',
        primaryBorderColor: '#E6E4D9',
        lineColor: '#6F6E69',
        secondaryColor: '#F2F0E5',
        tertiaryColor: '#FFFCF0',
        fontFamily: '-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif'
      }
    });
  </script>
{{ end }}
```

Notes on each design decision (do NOT add as comments to the file — the convention is no comment headers):
- `.Inner | safeHTML`: passes the diagram source through unchanged so Mermaid's runtime parser can read it (D-08).
- `.Page.Scratch` guard: ensures multiple `{{< mermaid >}}` blocks on the same page only emit the script tag once (per D-09 "loaded only on pages that use the shortcode" — applied at the page granularity using Hugo's per-page Scratch).
- `cdn.jsdelivr.net/npm/mermaid` URL: matches D-10 and the existing intervals-copilot post; intentionally unpinned to mirror the existing pattern (current convention does not pin Mermaid).
- `theme: 'base'`: required to enable `themeVariables` overrides in Mermaid (the `default`/`dark`/etc. presets ignore most overrides).
- The seven `themeVariables` hex values: copied verbatim from `themes/minimal/static/css/style.css` `:root` block (lines 4-18) per D-04. Mermaid's JS does not read CSS custom properties at runtime, so the hex values must be literal strings here.
- `fontFamily`: copied from `body` in style.css line 27 so SVG text matches site body text.
- Two-space indent inside `{{ if }}` block matches `themes/minimal/layouts/_default/baseof.html` template style.

Do NOT add a noscript fallback inside the shortcode (per PATTERNS.md, the noscript fallback is per-instance content; if needed, callers add their own `<noscript>` block in Markdown next to the shortcode).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → cdn.jsdelivr.net | Third-party JavaScript is fetched from a CDN and executed in the user's browser with full DOM access on tbohnstedt.cloud pages that include the shortcode. |
| Markdown author → Hugo template | Content authors write `.Inner` content that is passed through `safeHTML` and rendered as part of the page DOM (the diagram source becomes text inside `<div class="mermaid">`, then SVG via Mermaid's parser). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Mermaid CDN script load (`cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js`) | accept | Supply-chain risk identical to existing Leaflet (unpkg) and Plotly (cdn.plot.ly) loads in this site (climbing-routes post). Site is a static personal blog with no auth, no PII, no user accounts — compromised CDN script could deface pages but cannot steal credentials. SRI hash not added because (a) Mermaid is unpinned (`/npm/mermaid` resolves to latest) so the hash would change on every release and break the diagram, (b) project convention is unpinned CDN loads, (c) consistency with Leaflet/Plotly precedent. Documented as accepted risk. |
| T-01-02 | Information Disclosure | Mermaid CDN load reveals reader IP to jsdelivr.net | accept | Inherent to CDN model; consistent with existing third-party loads (Leaflet, Plotly, Instagram embed). Static personal blog with no privacy guarantees beyond the disabled Disqus/GA/X privacy settings already in hugo.toml. |
| T-01-03 | Injection / XSS via `.Inner | safeHTML` | Hugo shortcode template | accept | The `safeHTML` cast bypasses Hugo's HTML escaping, but the only authors of `.Inner` are the site owner (the user). There is no untrusted input — Markdown is authored by Timo and committed to git. Goldmark's `unsafe = true` (already site-wide) means raw HTML is allowed everywhere on the site, so this shortcode does not expand the attack surface. ASVS L1 input-validation controls do not apply (no user-submitted content). |
| T-01-04 | Denial of Service | Mermaid runtime crash on malformed diagram syntax | accept | A malformed diagram source produces a visible Mermaid error message in the rendered page, not a site outage. Author catches errors during local preview before commit. No mitigation needed beyond standard authoring review. |
| T-01-05 | Spoofing / Repudiation / Elevation of Privilege | N/A | n/a | No authentication, no user accounts, no privileged operations on the site. Categories not applicable to a static personal blog. |

No high-severity threats. All threats either accepted (with documented rationale consistent with existing site patterns) or N/A.
