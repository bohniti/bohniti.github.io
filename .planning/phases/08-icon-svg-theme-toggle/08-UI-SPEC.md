## Spacing Scale

The project does not declare a global spacing-token scale; it uses `rem`-based ad-hoc values throughout `style.css` (e.g. `1.5rem`, `0.5rem`, `2rem`). For this phase, the only spacing decisions are the toggle button's hit-area dimensions and its position in the nav row.

| Token | Value | Usage |
|-------|-------|-------|
| Tap target min | **44 px** (`min-width: 44px; min-height: 44px`) | `.theme-toggle` button shell — WCAG 2.5.5 / iOS HIG floor (D-06, REQ ICON-04) |
| Icon intrinsic | **24 px × 24 px** (`width="24" height="24"` on `<svg>`) | Both sun and moon SVGs — matches Lucide convention; visual size sits inside the 44 px shell (D-02) |
| Nav gap | **24 px** (`gap: 1.5rem` — existing `.site-nav` rule, `style.css:87`) | Spacing between nav links and the toggle — **not modified by this phase** |

**Exceptions:**
- **24 px icon inside 44 px shell** (D-06). The icon does NOT fill the button — the button shell absorbs the hit area via `display: inline-grid; place-items: center;`. This decouples icon size from hit area and makes the 44 CSS-px requirement self-documenting in the CSS.
- **Footer icons render at 18 px** (`themes/minimal/layouts/partials/footer.html:7,13`). Header toggle renders at **24 px** to match menu typography weight per CONTEXT.md `<code_context>`. Different sizes are intentional.

## Color

The project uses the locked Flexoki palette already declared at `style.css:4-33`. This phase declares **no new color tokens**. The icon recolors via `currentColor` cascading from the button's `color` property (mirroring the wordmark at `style.css:124-130` and the footer icons).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) — page surface | `var(--bg)` = `#FFFCF0` light / `#100F0F` dark | Page background; this phase does NOT modify it |
| Secondary (30%) — chrome text | `var(--text-secondary)` = `#6F6E69` light / `#878580` dark | **Default toggle icon stroke** — matches surrounding nav links at idle (`style.css:92`) |
| Accent (10%) — interaction | `var(--accent)` = `#AF3029` light / `#D14D41` dark | **Toggle icon stroke on `:hover`** and `:focus-visible` outline — matches existing `.theme-toggle:hover` rule at `style.css:113-115` |
| Destructive | not used in this phase | — |

**Accent reserved for** (10% rule, this phase only):
1. Toggle button SVG stroke on `:hover` (color transition from `var(--text-secondary)` → `var(--accent)`)
2. Toggle button `:focus-visible` outline ring (`outline: 2px solid var(--accent)` — already in `style.css:117-122`, kept verbatim per D-11)

**Hard-coded color prohibition (Pitfall 3 mitigation):**
- The `<svg>` markup MUST use `stroke="currentColor"` and `fill="none"`. Any literal `fill="#…"` or `stroke="#…"` in the path data is a contract violation. CI / reviewer grep: `grep -E 'fill="#|stroke="#"' themes/minimal/layouts/partials/header.html` must return zero results.
- The two CSS variables this phase touches are `var(--text-secondary)` (idle) and `var(--accent)` (hover/focus). No hex literals in CSS for the toggle.

**Theme-color meta (Pitfall 22 mitigation, D-12):** The existing `<meta name="theme-color">` sync stays as-is — `#FFFCF0` (light) / `#100F0F` (dark). The end-of-body click handler must continue writing this. Single source of truth: `documentElement.dataset.theme`.

## Copywriting Contract

Although the toggle has no visible text, copywriting still defines the accessible name announced by screen readers (REQ ICON-03) and the action-oriented contract from D-08.

| Element | Copy |
|---------|------|
| Visible button text | **(none — removed per D-11)** |
| `aria-label` when current theme is **light** | **`Switch to dark mode`** (icon shown: sun; clicking will go to dark) |
| `aria-label` when current theme is **dark** | **`Switch to light mode`** (icon shown: moon; clicking will go to light) |
| `aria-pressed` when light is active | `false` (D-09; preserved from v2.0) |
| `aria-pressed` when dark is active | `true` (D-09; preserved from v2.0) |
| Server-rendered initial `aria-label` | `Switch to dark mode` (matches the IIFE's "no preference saved" fallback; the end-of-body handler reconciles to actual resolved theme on load — D-08, mirrors existing `textContent` reconciliation at `baseof.html:43-45`) |
| Server-rendered initial `aria-pressed` | `false` (matches existing convention at `header.html:11`) |
| Empty state | n/a — toggle is always present |
| Error state | n/a — toggle has no failure mode (degrades gracefully if `localStorage` throws; the existing `try/catch` at `baseof.html:52` is preserved) |
| Destructive confirmation | n/a — toggling theme is non-destructive and instantly reversible |

**Action-orientation contract (D-03, D-08):** The **icon depicts the current theme** (sun = light is active; moon = dark is active). The **`aria-label` describes the target action** (what clicking will do). This is the GitHub-style pattern. Do not invert: a sun icon with `aria-label="Switch to light mode"` would be a contract violation — the user is already in light mode.

**Tone:** Plain imperative. No marketing copy, no exclamation marks, no emoji (PROJECT.md exclusion: "no emoji `☀️ 🌙`").

### Button Markup Contract

```html
<button type="button" class="theme-toggle" aria-pressed="false" aria-label="Switch to dark mode">
  <!-- sun SVG (above) -->
  <!-- moon SVG (above) -->
</button>
```

Attribute requirements:
- `type="button"` — preserved from existing markup (D-decisions §Claude's Discretion); prevents implicit form submission edge-cases.
- `class="theme-toggle"` — selector kept for JS query (`document.querySelector('.theme-toggle')`).
- `aria-pressed` — initial value `"false"`; reconciled to actual state by the end-of-body IIFE on load.
- `aria-label` — initial value `"Switch to dark mode"`; reconciled to actual state by the end-of-body IIFE on load.
- Both inner `<svg>` elements MUST carry `aria-hidden="true"` (the button is the labeled control — Sara Soueidan accessible-icon-buttons pattern).

### Visibility Swap Mechanism (D-04, D-05; Pitfall 1 mitigation)

CSS-only swap, keyed off `[data-theme]` on `<html>`. The head IIFE at `baseof.html:11-23` sets `dataset.theme` **before** the stylesheet loads — so the correct icon is the only visible one at first paint. Zero FOUC.

```css
.theme-toggle {
  display: inline-grid;
  place-items: center;
  min-width: 44px;
  min-height: 44px;
  background: transparent;
  border: 0;
  padding: 0;
  cursor: pointer;
  color: var(--text-secondary);
}

.theme-toggle .icon-sun,
.theme-toggle .icon-moon {
  grid-area: 1 / 1;            /* stack in the same grid cell — no layout reflow on swap */
  display: block;
}

/* Light theme: show sun, hide moon */
:root[data-theme="light"] .theme-toggle .icon-moon,
:root:not([data-theme="dark"]) .theme-toggle .icon-moon {
  opacity: 0;
  pointer-events: none;
}

/* Dark theme: show moon, hide sun */
:root[data-theme="dark"] .theme-toggle .icon-sun {
  opacity: 0;
  pointer-events: none;
}

.theme-toggle:hover {
  color: var(--accent);
}

/* Focus ring — kept verbatim from existing rule, applies to button shell */
.theme-toggle:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 2px;
  border-radius: 2px;
}
```

Note the `:root:not([data-theme="dark"])` selector covers the rare case where the head IIFE has not yet written `data-theme` — defensive against CSS-loading-before-IIFE (which the architecture prevents, but the selector costs nothing).

### Transition Contract (D-07; REQ ICON-05; Pitfall 9 mitigation)

**Opacity-only cross-fade, ≤ 150 ms, ease.** No rotation, no scale, no translate. Wrapped in `@media (prefers-reduced-motion: no-preference)` — exact mirror of the existing body-transition pattern at `style.css:49-56`.

```css
@media (prefers-reduced-motion: no-preference) {
  .theme-toggle .icon-sun,
  .theme-toggle .icon-moon {
    transition: opacity 150ms ease;
  }
}
```

When reduced-motion is set, opacity transitions are simply not declared — the swap is instant. Do NOT add a "tasteful" 50 ms fallback; spec intent is **zero unrequested motion**.

Rationale for opacity-only (per D-07): rotation adds motion that does not improve comprehension and risks vestibular discomfort. Flexoki/Kindle/Obsidian aesthetic favors stillness.

### Hit Target Contract (D-06; REQ ICON-04; Pitfall 2 mitigation)

- Button shell: **`min-width: 44px; min-height: 44px`** (explicit, in the `.theme-toggle` rule above).
- Icon: **24 px × 24 px** (intrinsic, on the `<svg>` element).
- Centering: `display: inline-grid; place-items: center;` — no padding needed.
- Verification gate: `getBoundingClientRect()` of `.theme-toggle` in Chrome DevTools mobile emulation must return both `width >= 44` and `height >= 44`. Lighthouse Accessibility audit must NOT flag "Touch targets do not have sufficient size or spacing."
- **Mobile reflow note:** `style.css:399` switches the header to `flex-direction: column` at `< 600 px`. Adjacent-link mistap is moot in column layout; desktop case is the load-bearing one. Confirm during HUMAN-UAT that the column-layout button still renders at 44 px (no stretching).

### JS Contract (D-10; Pitfall 22 mitigation)

The end-of-body IIFE at `baseof.html:34-56` is rewritten. **Single source of truth: `documentElement.dataset.theme`.** JS never touches the SVGs.

Responsibilities (preserved from v2.0):
1. Read `documentElement.dataset.theme` to determine current state.
2. On click: write `next` value to `documentElement.dataset.theme`.
3. Persist: `localStorage.setItem('theme', next)` inside `try/catch` (Safari private mode safety).
4. Sync: `meta[name="theme-color"]` content attribute — `#100F0F` for dark, `#FFFCF0` for light.
5. Reconcile on load: set initial `aria-pressed` and `aria-label` to match the theme the head IIFE resolved.

Responsibility **changed** from v2.0:
- Replace `toggle.textContent = next === 'dark' ? 'Light' : 'Dark'` with `toggle.setAttribute('aria-label', next === 'dark' ? 'Switch to light mode' : 'Switch to dark mode')`.

Responsibility **removed** from v2.0:
- `toggle.textContent = …` lines (button no longer renders text).

Responsibility explicitly **NOT added** (Pitfall 22):
- Imperative SVG class toggling. Icon visibility is 100% CSS.
- Imperative `display: none / block` on either SVG. Stays opacity-driven.

### State Inventory

| State | Visual | a11y |
|-------|--------|------|
| **Idle, light** | Sun icon visible, `var(--text-secondary)` stroke | `aria-pressed="false"`, `aria-label="Switch to dark mode"` |
| **Idle, dark** | Moon icon visible, `var(--text-secondary)` stroke | `aria-pressed="true"`, `aria-label="Switch to light mode"` |
| **Hover (any theme)** | Visible icon's stroke transitions to `var(--accent)` over 150 ms | unchanged |
| **Focus-visible (any theme)** | 2 px `var(--accent)` outline at 2 px offset, 2 px border-radius | unchanged |
| **Active / clicked** | Theme swaps; icon cross-fades over 150 ms (or instant if reduced-motion) | `aria-pressed` and `aria-label` flip atomically with `dataset.theme` write |
| **Reduced-motion, any** | Same as default but no opacity transition; swap is instant | unchanged |

### Accessibility Contract Summary (REQ ICON-03)

- Accessible name: `aria-label` on `<button>` (action-oriented).
- State: `aria-pressed` on `<button>` (current state).
- Inner SVGs: `aria-hidden="true"` (decorative — name comes from button).
- Hit target: ≥ 44 × 44 CSS-px (WCAG 2.5.5).
- Focus indicator: visible 2 px outline (existing rule preserved).
- Reduced motion: respected via `@media (prefers-reduced-motion: no-preference)` gating.
- No flash of wrong icon: server-rendered both SVGs + CSS-keyed visibility on `[data-theme]` set in head IIFE before stylesheet loads.
