**Step 3 — One-shot exiftool source-side scrub (D-13 prong 1, hardened per RESEARCH § Field Coverage Audit):**

The CONTEXT-locked invocation strips GPS, Make, Model, SerialNumber, InternalSerialNumber. RESEARCH § Field Coverage Audit recommends hardening with `-Serial*=` (collapses LensSerialNumber, BodySerialNumber too) and adding `-OwnerName= -Artist= -Copyright=` for cheap insurance. Apply the hardened invocation:

```bash
cd content/gallery/photos

exiftool \
  -GPS:All= \
  -Make= -Model= \
  -Serial*= \
  -OwnerName= -Artist= -Copyright= \
  -overwrite_original \
  ./*

cd - > /dev/null
```

Expected output: one line per file in the form `1 image files updated` for files that had matching tags, or no output / `0 image files updated` for the UUID-named files that were already pre-stripped (per RESEARCH § Empirical Findings: 7 of 18 are EXIF-empty UUID files).

Notes on flags (verified RESEARCH § exiftool Invocation Reference):
- `-TAG=` (no value) deletes the tag.
- `-GPS:All=` deletes ALL tags in the GPS group (equivalent to `-GPS:*=`).
- `-Serial*=` uses exiftool's wildcard-in-tag-name to match `SerialNumber`, `InternalSerialNumber`, `LensSerialNumber`, `BodySerialNumber` in one pass.
- `-overwrite_original` writes the scrub in place (default would create `_original` backup files we don't want).
- `./*` is shell-expanded by zsh/bash; matches all 18 files in the current dir (no subdirs to recurse into).
