<!-- Render-image hook contract (this plan creates it) -->
The hook receives, for each markdown image reference `![alt](src "title")`:
- `.Destination` → the `src` string (e.g., `"images/portrait.jpg"`)
- `.Text` → the alt text (e.g., `"Portrait of Timo"`)
- `.Title` → the third-tuple title literal (e.g., `"hero"`, `"grid"`, or `""` for no title)
- `.Page` → the page on which the image is rendered (used for `.Resources.GetMatch`)

The hook must:
1. Try `$resource := .Page.Resources.GetMatch .Destination`
2. If found → process via `image.Process "fill WxH Smart webp qN"` with W/H/N chosen by `.Title` keyword (hero=480x600 q80, grid=400x300 q75, default=800x600 q78)
3. Emit `<img>` with `src=$processed.RelPermalink`, explicit `width`/`height` from processed dims, alt from `.Text`, and per-keyword class + loading + fetchpriority attributes
4. If lookup returns nil → fall through to `<img src="{{ .Destination | safeURL }}" alt="{{ .Text }}">` so legacy blog post images (and external URLs) still render
</interfaces>

Create the file `themes/minimal/layouts/_default/_markup/render-image.html` with the verbatim source below (D-19, whitespace-trimmed). Hugo automatically picks up files at this path as the global render-image hook for ALL markdown image references site-wide. The hook resolves bundle-relative paths via `.Page.Resources.GetMatch`, processes to WebP via `image.Process` with `.Title`-keyword-switched dimensions, and falls back to a passthrough `<img>` for non-bundle references (matters for existing blog post images per D-20).

Exact file contents (paste verbatim, no extra lines, no trailing whitespace):

```go-html-template
{{- $resource := .Page.Resources.GetMatch (printf "%s" .Destination) -}}
{{- if $resource -}}
  {{- $title := .Title | default "" -}}
  {{- $isHero := eq $title "hero" -}}
  {{- $isGrid := eq $title "grid" -}}
  {{- $processed := "" -}}
  {{- if $isHero -}}
    {{- $processed = $resource.Process "fill 480x600 Smart webp q80" -}}
  {{- else if $isGrid -}}
    {{- $processed = $resource.Process "fill 400x300 Smart webp q75" -}}
  {{- else -}}
    {{- $processed = $resource.Process "fill 800x600 Smart webp q78" -}}
  {{- end -}}
  <img src="{{ $processed.RelPermalink }}"
       width="{{ $processed.Width }}"
       height="{{ $processed.Height }}"
       alt="{{ .Text }}"
       {{- if $isHero }} class="about-hero-img" loading="eager" fetchpriority="high"{{ else if $isGrid }} class="about-grid-item" loading="lazy" decoding="async"{{ else }} loading="lazy" decoding="async"{{ end -}}>
{{- else -}}
  <img src="{{ .Destination | safeURL }}" alt="{{ .Text }}"{{ with .Title }} title="{{ . }}"{{ end }}>
{{- end -}}
```

Whitespace-control notes:
- The `{{-` / `-}}` trims keep the rendered HTML compact so the CSS Grid layout doesn't get stray text nodes.
- The `<img>` opening tag spans multiple lines for readability but emits as a single tag because of the `-` trims around the conditional class/loading attributes.
- The `else` (fallback) arm passes the destination through `safeURL` to preserve any pre-encoded characters (e.g., `%20` in legacy blog post image references like `images/Chaotic%20timeline.png` per RESEARCH § item 5).

Why this exact shape:
- D-06 locks the three-arm switch on the `"hero"` / `"grid"` / default title keyword.
- D-07 locks `loading="eager" fetchpriority="high"` on the hero arm only (single above-fold image).
- D-19 locks the whitespace trimming AND the suppression of the `title="..."` attribute on in-bundle resources (the keyword is consumed as a sizing hint, not leaked to the rendered DOM).
- The fallback arm preserves `title` ONLY for non-bundle resources — this matches Bep's `portable-hugo-links` reference pattern and keeps existing blog post hover-titles intact.

DO NOT touch any other layout files in this task. The hook is a single new file; the rest of the layout chain (baseof.html, single.html) is unchanged.
