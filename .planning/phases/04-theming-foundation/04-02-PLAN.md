    // Theme bootstrap — reads localStorage key 'theme', falls back to OS prefers-color-scheme.
    (function () {
      let theme;
      try { theme = localStorage.getItem('theme'); } catch (_) { /* Safari private mode */ }
      if (theme !== 'light' && theme !== 'dark') {

      const meta = document.querySelector('meta[name="theme-color"]');

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser localStorage → IIFE → DOM dataset | localStorage is user-controlled (DevTools / extensions can write arbitrary values). IIFE reads from it and writes to `document.documentElement.dataset.theme` and `<meta name="theme-color">`. |
| OS prefers-color-scheme → IIFE | UA-controlled boolean; not a tampering surface but a fallback signal. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering / Injection | localStorage `theme` value | mitigate | The IIFE accepts ONLY the literal strings `'dark'` and `'light'`. Any other value (`'<script>...'`, `'system'`, empty string, JSON object stringified) fails the `theme !== 'light' && theme !== 'dark'` check and triggers fallback to `matchMedia`. The sanitized value is what gets written to `dataset.theme`. Even if a malicious value could pass through, it would set `data-theme` to an attribute selector that matches no CSS rule (no XSS surface — `data-theme` is not interpolated into HTML; it is read by attribute selector only). |
| T-04-06 | Tampering | `<meta name="theme-color">` content | mitigate | The IIFE writes ONLY one of two locked literal strings: `'#100F0F'` or `'#FFFCF0'`. The choice is determined by the already-sanitized `theme` variable, so a tampered localStorage cannot inject arbitrary content into the meta tag. |
| T-04-07 | Information Disclosure | Inline IIFE source | accept | The IIFE source is visible in HTML view-source (it must be — it runs before any external script can fetch). It reveals the project uses localStorage with key `'theme'` and OS preference fallback. This is intentionally public; no secrets are present. |
| T-04-08 | Denial of Service | localStorage quota / Safari private mode | mitigate | The IIFE wraps localStorage access in try/catch. Safari private mode throws on `getItem`; the catch swallows it and falls through to `matchMedia`. Quota exceeded is not a read-side concern. |
| T-04-09 | Spoofing | matchMedia result | accept | OS preference is a UA-trusted signal. No mitigation needed. |

Severity: low across the board. Highest-impact threat (T-04-05) is mitigated structurally by the input-allowlist sanitization built into the IIFE.
</threat_model>
