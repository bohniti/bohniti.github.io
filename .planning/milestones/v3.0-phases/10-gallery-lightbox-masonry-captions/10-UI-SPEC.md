# Phase 10 — UI Design Contract

> Visual and interaction contract for the GALLERY (Lightbox + Masonry + Captions) phase. Generated by gsd-ui-researcher; six dimensions encoded as falsifiable gates the gsd-ui-checker can grep/verify against `style.css`, `gallery/single.html`, and `lightbox.js`.

This phase ships **NO new design tokens, NO new fonts, NO new color hexes** beyond the single rgba pair `(16, 15, 15, 0.85)` and `(16, 15, 15, 0.6)` already locked in CONTEXT D-12 / D-13. Everything else routes through existing Flexoki CSS custom properties (`--bg`, `--bg-secondary`, `--text`, `--text-secondary`, `--border`, `--accent`, `--radius-soft`).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Hugo + vanilla CSS/JS, no component framework) |
| Preset | not applicable |
| Component library | none (native `<dialog>` + CSS column-count) |
| Icon library | hand-authored inline SVG, Lucide visual language (NOT installed as a package — anti-feature per CONTEXT) |
| Font | system stack (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, sans-serif`) — inherited from `body`; no web fonts |

## Spacing Scale

Declared values (rem-based; `1rem = 17px` desktop / `16px` mobile per existing `html { font-size }` rule):

| Token | Value | Usage in Phase 10 |
|-------|-------|-------------------|
| xs | 0.25rem (4px) | (unused — reserved) |
| sm | 0.5rem (8px) | Lightbox button padding (icon center) |
| md | 1rem (16px) | `column-gap`, `.gallery-item { margin-bottom }`, button edge offset from dialog |
| lg | 1.5rem (24px) | Caption top spacing below image, dialog inner padding |
| xl | 2rem (32px) | (unused — reserved) |
| 2xl | 3rem (48px) | (unused — reserved) |
| reserved | `calc(100vh - 8rem)` | Lightbox image `max-height` (D-25) — NOT a spacing token; encodes vertical chrome budget |

Exceptions:
- **44×44 minimum hit-target on all three lightbox buttons** (close, prev, next) — WCAG 2.5.5 / iOS HIG. May use `min-width: 44px; min-height: 44px; display: inline-grid; place-items: center;` (Phase 8 `.theme-toggle` precedent at `style.css:103-116`). The inner SVG remains 24×24; hit area decouples from icon size.
- Dialog button edge offsets (`top`, `right`, `left`) inside `dialog#gallery-lightbox` may be `0.5rem`–`1rem`; planner picks within that range, NOT outside. Goal: buttons sit visually within the dialog's `--radius-soft`-rounded inner bounds without overlapping the image.

## Typography

Phase 10 introduces **NO new type sizes or weights** — caption typography sits on the existing site `body` defaults, with one explicit override.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (inherited) | 1rem (17px desktop / 16px mobile) | 400 | 1.7 (from `body`) |
| Lightbox caption (`.gallery-lightbox-caption`) | 0.95rem | 400 | 1.5 |
| Lightbox button SVG (no text) | n/a (24×24 viewBox, 2px stroke, currentColor) | n/a | n/a |
| Page header `h1` (existing, unchanged) | 1.6rem (`.page-title`) | 700 | 1.3 |

Caption rules:
- Color: `var(--text-secondary)` — secondary surface inside an attention-grabbing modal; primary photo wins visual hierarchy.
- `max-width: 60ch` — readable measure; prevents captions stretching across a wide dialog on ≥1200px viewports.
- `text-align: center` — visual symmetry with centered image; matches `tylerkarow.com/gallery` reference aesthetic.
- `margin: 1rem auto 0;` — top spacing from image baseline; horizontal `auto` pairs with `max-width` for centering.
- Empty figcaption (caption-less photos per D-02) renders zero DOM (`{{ with $photo.Params.caption }}…{{ end }}` template guard); no empty `<figcaption>` element ships.

## Color

Phase 10 uses ONLY existing Flexoki tokens plus the locked backdrop rgba pair.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` | Gallery page background, lightbox dialog inner background |
| Secondary (30%) | `var(--bg-secondary)` | (unused in lightbox; reserved if planner tints button shells) |
| Accent (10%) | `var(--accent)` | `:focus-visible` outline ONLY — on `.gallery-item`, `.gallery-lightbox-close`, `.gallery-lightbox-prev`, `.gallery-lightbox-next` |
| Text | `var(--text)` | Lightbox button SVG `currentColor` source (default state) |
| Text secondary | `var(--text-secondary)` | Caption color, button hover-out state |
| Border | `var(--border)` | (unused in lightbox; reserved) |
| Backdrop fallback | `rgba(16, 15, 15, 0.85)` | `dialog#gallery-lightbox::backdrop` default (no `backdrop-filter` support) |
| Backdrop with blur | `rgba(16, 15, 15, 0.6)` + `backdrop-filter: blur(12px)` | `@supports (backdrop-filter: blur(12px))` enhancement |
| Destructive | n/a | No destructive actions in this phase |

Accent reserved for:
1. `:focus-visible` outline on `.gallery-item` thumbnails (matches Phase 8 `.theme-toggle:focus-visible` pattern at `style.css:152-157`)
2. `:focus-visible` outline on the three `.gallery-lightbox-{close,prev,next}` buttons (same pattern, same `2px solid var(--accent)`, same `outline-offset: 2px`)
3. `.gallery-lightbox-{close,prev,next}:hover` color shift (transitions `color: var(--text)` → `color: var(--accent)`, ≤150ms, ONLY inside `@media (prefers-reduced-motion: no-preference)`) — matches Phase 8 `.theme-toggle:hover` pattern at `style.css:148-150`

Accent is NEVER applied to:
- Backdrop tint (theme-invariant rgba; D-13)
- Caption text (uses `--text-secondary`)
- Button default state (uses `--text` via `currentColor`)
- Image borders or frames (none exist; photos render flush with no border)

## Copywriting Contract

Phase 10 adds three button labels, one dialog accessible name (dynamic), and zero empty/error/destructive copy (gallery has no empty state — if no photos exist, the page bundle has no `[[resources]]` and the template renders nothing visible; this is acceptable).

| Element | Copy |
|---------|------|
| Primary CTA (thumbnail wrapper `aria-label`) | `Open photo {N} of {Total} at full size` (existing v2.0 pattern at `gallery/single.html:14`; preserved per D-28) |
| Empty state heading | n/a — no photos = no rendered gallery grid (acceptable; gallery is curated, never empty) |
| Empty state body | n/a |
| Error state | n/a — no failure modes surface in UI (Hugo build-failure happens at deploy time, not in browser) |
| Destructive confirmation | n/a — no destructive actions |
| Lightbox close button `aria-label` | `Close gallery` (target action; matches Phase 8 ICON-03 action-oriented pattern) |
| Lightbox prev button `aria-label` | `Previous photo` |
| Lightbox next button `aria-label` | `Next photo` |
| Dialog `aria-label` (dynamic) | `Photo {N} of {Total}` — set by JS on open and on each prev/next nav (D-17). Initial server-rendered value: `Photo viewer` (placeholder before first open) |
| Lightbox image `alt` | `{photo.Params.alt}` — author-provided per `[[resources]]` block (D-03 — REQUIRED field). Thumbnail `<img alt="">` stays empty (decorative; the wrapping `<a aria-label>` provides the accessible name per D-28) |
| Caption text | `{photo.Params.caption}` — author-provided plain text, 1–2 sentences, no markdown processing (anti-feature). Empty/missing caption → zero DOM (D-02) |

Tone rules (carry-forward from Phase 8 ICON-03 pattern):
- Buttons describe the **target action**, not the current state. "Close gallery" not "Gallery is open". "Previous photo" not "Go back".
- Captions tell a 1–2 sentence story; not technical specs, not EXIF data, not GPS (anti-feature per CONTEXT specifics).

## Six Dimensional Visual Gates

Concrete grep-able contracts for `gsd-ui-checker`. Each gate ships as a verifiable assertion against the working tree.

### Gate 1 — Page-level Skeleton (Masonry Grid)

**Files:** `themes/minimal/static/css/style.css` `/* === Gallery === */` block (currently lines 311–339); `themes/minimal/layouts/gallery/single.html`.

Falsifiable assertions:
1. `style.css` must NOT contain `display: grid` inside any `body.page-gallery .gallery-grid` rule. (Replaces v2.0 grid with column-count.)
2. `style.css` must contain a `body.page-gallery .gallery-grid { column-count: 3; column-gap: 1rem; }` rule (or equivalent default at `min-width >= 900px`).
3. `style.css` must contain a `body.page-gallery .gallery-grid { column-count: 2; }` rule under `@media (min-width: 600px) and (max-width: 899px)` (or equivalent breakpoint range).
4. `style.css` must contain a `body.page-gallery .gallery-grid { column-count: 1; }` rule inside the existing `@media (max-width: 600px)` block.
5. `style.css` must contain `body.page-gallery .gallery-item { break-inside: avoid; margin-bottom: 1rem; }` (or `margin-bottom` ≤ `1rem`, ≥ `0.75rem`).
6. `gallery/single.html` `range` clause must read `range $idx, $photo := (sort (.Resources.Match "photos/*") "Params.weight")` or equivalent weight-sorted iteration.
7. `gallery/single.html` thumbnail directive must be `Resize "600x webp q75"` (NOT `fill 600x400 Smart webp q75`).
8. Every `<a class="gallery-item">` must emit `data-caption="{{ $photo.Params.caption }}"` AND `data-alt="{{ $photo.Params.alt }}"` attributes (in addition to existing `href`, `aria-label`).

### Gate 2 — Lightbox Dialog Skeleton

**Files:** `themes/minimal/layouts/gallery/single.html` (bottom of `{{ define "main" }}`); `themes/minimal/static/js/lightbox.js` (NEW).

Falsifiable assertions:
1. `gallery/single.html` must contain a single `<dialog id="gallery-lightbox" aria-label="Photo viewer">` element (D-09 — one dialog DOM, not 18).
2. The dialog must contain, in this order: a `<button class="gallery-lightbox-close">`, a `<button class="gallery-lightbox-prev">`, an `<img class="gallery-lightbox-img">`, a `<figcaption class="gallery-lightbox-caption">`, a `<button class="gallery-lightbox-next">` (D-20).
3. Each button must have a hand-authored inline SVG child (NOT an `<img>` reference, NOT a `<use href>` to an external sprite). SVG attributes must include `viewBox="0 0 24 24"`, `fill="none"`, `stroke="currentColor"`, `stroke-width="2"`, `aria-hidden="true"`. Dimensions match `partials/footer.html` icon pattern (intrinsic `width="24" height="24"`).
4. Each button must have a static `aria-label` matching the Copywriting Contract above (`Close gallery` / `Previous photo` / `Next photo`).
5. `gallery/single.html` must include `<script src="{{ "js/lightbox.js" | absURL }}" defer></script>` at the bottom of `{{ define "main" }}` — page-scoped, NOT loaded from `baseof.html` (D-14).
6. `lightbox.js` must call `dialog.showModal()` (NOT `dialog.show()`, NOT `dialog.removeAttribute('hidden')`) — native focus trap depends on `showModal()` (D-08).
7. `lightbox.js` must set `dialog.setAttribute('aria-label', ...)` on each open and on each prev/next navigation (D-17).
8. `lightbox.js` must set `document.body.style.overflow = 'hidden'` on dialog open and restore the prior value on close (D-11).
9. `lightbox.js` must read full-image URL from `a.href` (NOT from inner `<img src>`) — Pitfall 7 (D-15 step 1).
10. `lightbox.js` must NOT preload next/prev images (anti-feature; deferred GALLERY-FUT-02).

### Gate 3 — Backdrop Visual

**Files:** `themes/minimal/static/css/style.css` `/* === Gallery — Lightbox === */` block (NEW).

Falsifiable assertions:
1. `style.css` must contain a default rule (outside `@supports`):
   ```css
   body.page-gallery dialog#gallery-lightbox::backdrop {
     background: rgba(16, 15, 15, 0.85);
   }
   ```
2. `style.css` must contain a progressive-enhancement rule:
   ```css
   @supports (backdrop-filter: blur(12px)) {
     body.page-gallery dialog#gallery-lightbox::backdrop {
       background: rgba(16, 15, 15, 0.6);
       backdrop-filter: blur(12px);
       -webkit-backdrop-filter: blur(12px);
     }
   }
   ```
3. The backdrop background MUST be theme-invariant (single rgba pair; no `:root[data-theme="dark"]` override) per D-13.
4. The blur radius MUST be exactly `12px` (locked by ROADMAP success criterion #3).
5. `-webkit-backdrop-filter` prefix MUST appear alongside the unprefixed property (Safari <17 support; D-12).
6. NO `box-shadow` on the dialog or backdrop (Pitfall 16 — Flexoki-flat, no shadows).

### Gate 4 — Buttons (Visual + Hit Target)

**Files:** `themes/minimal/static/css/style.css` `/* === Gallery — Lightbox === */` block (NEW); `themes/minimal/layouts/gallery/single.html` (button markup).

Falsifiable assertions:
1. Every `.gallery-lightbox-{close,prev,next}` rule must declare `min-width: 44px;` AND `min-height: 44px;` (WCAG 2.5.5 / iOS HIG; matches Phase 8 `.theme-toggle` pattern at `style.css:107-108`).
2. Buttons must be `display: inline-grid; place-items: center;` (decouples 24×24 SVG from 44×44 hit area; Phase 8 precedent at `style.css:105-106`).
3. Button reset properties must be present: `background: transparent; border: 0; padding: 0; cursor: pointer;` (Phase 8 precedent at `style.css:110-113`).
4. Default button color: `color: var(--text);` — drives inner SVG `currentColor` cascade. NOT `var(--accent)` (accent is reserved for focus and hover only).
5. Hover state: `color: var(--accent);` — wrapped in `@media (prefers-reduced-motion: no-preference) { transition: color 150ms ease; }` (Pitfall 9; D-22). The base `:hover { color: var(--accent); }` rule itself is OUTSIDE the media query (color change is not motion); only the `transition` is wrapped.
6. `:focus-visible` state: `outline: 2px solid var(--accent); outline-offset: 2px;` — exact same triple as Phase 8 `.theme-toggle:focus-visible` (`style.css:153-156`). Optional `border-radius: var(--radius-soft);` to match Phase 9 token aesthetics on the focus ring.
7. Button positioning inside dialog: `position: absolute;` for prev/next/close (close at top-right, prev at left-middle, next at right-middle). Z-index above image but below `::backdrop` (no explicit z-index needed; native dialog top-layer handles stacking).
8. SVG inside button MUST have `aria-hidden="true"` (the `aria-label` on the parent button provides the accessible name; Sara Soueidan accessible-icon-button pattern).
9. Forbidden: emoji (e.g. `×`, `←`, `→` Unicode glyphs as button content). The visual MUST be hand-authored SVG paths only.

### Gate 5 — Caption Typography

**Files:** `themes/minimal/static/css/style.css` `/* === Gallery — Lightbox === */` block.

Falsifiable assertions:
1. `style.css` must contain a rule matching:
   ```css
   body.page-gallery .gallery-lightbox-caption {
     font-size: 0.95rem;
     line-height: 1.5;
     color: var(--text-secondary);
     max-width: 60ch;
     margin: 1rem auto 0;
     text-align: center;
   }
   ```
   Tolerances: `font-size` may be `0.9rem`–`1rem`; `line-height` may be `1.4`–`1.6`; `max-width` may be `48ch`–`72ch`; `margin-top` may be `0.75rem`–`1.25rem`. Other properties exact.
2. Color MUST be `var(--text-secondary)` (NOT `var(--text)`, NOT `var(--accent)`).
3. NO `font-style: italic;` (caption is plain prose, not a quotation).
4. NO `font-family` override (inherits system stack from `body`).
5. NO `font-weight` override (inherits 400 from `body`).
6. The figcaption element MUST be conditionally rendered via Hugo `{{ with $photo.Params.caption }}…{{ end }}` — empty captions ship zero DOM (D-02).

### Gate 6 — Image Sizing, Reduced-Motion, Theme Coverage

**Files:** `themes/minimal/static/css/style.css` `/* === Gallery — Lightbox === */` block.

Falsifiable assertions:
1. `style.css` must contain:
   ```css
   body.page-gallery .gallery-lightbox-img {
     max-width: 100%;
     max-height: calc(100vh - 8rem);
     object-fit: contain;
     display: block;
     margin: 0 auto;
   }
   ```
   Exact values for `max-width`, `max-height`, `object-fit` (D-25). `margin: 0 auto;` centers within dialog.
2. NO `width: 100%` declaration on `.gallery-lightbox-img` (would distort aspect ratio).
3. Every `<img>` in the rendered gallery (thumbnail AND lightbox) must ship explicit `width` AND `height` HTML attributes from Hugo's processed image (`width="{{ $thumb.Width }}" height="{{ $thumb.Height }}"`) — REQ GALLERY-08, CLS < 0.1 (D-24).
4. Thumbnail rule `body.page-gallery .gallery-img { width: 100%; height: auto; display: block; }` retained from v2.0 (existing `style.css:334-339`); pair with the per-photo intrinsic `width`/`height` for implicit `aspect-ratio` reservation.
5. Optional thumbnail `border-radius: var(--radius-soft);` MAY apply (planner's discretion within D-21 prefix); current v2.0 value of `border-radius: 4px` SHOULD be raised to `var(--radius-soft)` to match Phase 9 site-wide soft-corner aesthetic. Lightbox image itself does NOT need rounded corners (the modal is the visual frame; photo content fills it).
6. ALL transitions in the new `/* === Gallery — Lightbox === */` block MUST be wrapped in `@media (prefers-reduced-motion: no-preference) { … }` (D-22; Pitfall 9). Specifically: button color hover transitions; backdrop fade-in (if any); dialog open scale (if any). Reduced-motion users see instant open/close.
7. Transition durations MUST be `≤ 150ms` (locked by REQUIREMENTS Out of Scope: "keep transitions to ≤ 150 ms CSS").
8. The `/* === Gallery — Lightbox === */` block MUST NOT contain ANY `:root[data-theme="dark"]` override rules. All theme-specific values come from CSS custom properties (`--text`, `--text-secondary`, `--accent`, `--bg`, `--bg-secondary`, `--border`) which already cascade per-theme. The backdrop rgba is theme-invariant by design (D-13).
9. Verify dark-theme legibility manually (HUMAN-UAT): caption `var(--text-secondary)` on `var(--bg)` lightbox interior remains ≥ 4.5:1 contrast in BOTH `:root` and `:root[data-theme="dark"]`. (Flexoki tokens are pre-vetted; no new colors introduced — gate is satisfied by token reuse.)
10. ALL new lightbox/masonry CSS rules MUST be prefixed `body.page-gallery` (D-21; Pitfall 17). Forbidden generic selectors: `.lightbox`, `.modal`, `.close`, `.prev`, `.next`, `.dialog`, `dialog#gallery-lightbox` (without the body prefix).
